📖 How to Play

🎮 Single Player

Use ↑ ↓ ← → arrow keys (or swipe on mobile) to slide all tiles. When two tiles with the same number touch, they merge into one. Reach 2048 to win!

👥 Two Player

  • Player 1 uses W A S D
  • Player 2 uses ↑ ↓ ← →
  • Both boards run simultaneously. Winner is whoever has the highest score when both get game over, or whoever reaches 2048 first.

🤖 Me vs AI

  • You play with arrow keys on the left board. The AI auto-plays the right board.
  • Easy — AI picks random valid moves
  • Medium — Heuristic: prefers merges, empty cells, corner placement
  • Hard — Game tree search (3-ply lookahead)

🔧 Controls

  • Undo — Reverts to previous state (Stack pop)
  • Save — Saves current state to browser storage
  • Load — Restores last saved state
  • Reset — Starts a fresh game
  • DSA — Opens full data structures dashboard

🔬 About the Project

Data Structures Used

Stack

Powers the Undo system. Each move pushes the board state; undo pops it back. LIFO ensures the most recent move is undone first.

Push/Pop: O(1) | Space: O(n)

Queue

Logs all moves in FIFO order. Used for action history replay and turn management in two-player mode.

Enqueue/Dequeue: O(1) | Space: O(n)

Doubly Linked List

Stores move history allowing bidirectional traversal — navigate forward and backward through game history.

Insert: O(1) | Navigate: O(1) | Space: O(n)

Game Tree

Used by the Hard AI. Builds an N-ary tree of future board states up to depth 3 and picks the highest-scoring path.

Build: O(b^d) | Space: O(b^d)

Graph (Adjacency List)

Tracks every visited board state as a node, with moves as directed edges. Detects revisited states and shows game exploration.

Add: O(1) | Space: O(V+E)

BST (Binary Search Tree)

Maintains the sorted leaderboard. In-order traversal produces ranked scores automatically. No manual sorting needed.

Insert/Search: O(log n) avg | Traverse: O(n)

Technical Stack

Built with pure HTML5, CSS3, and Vanilla JavaScript — no frameworks. Architecture uses OOP classes with separation of concerns across 9 JavaScript modules. Persistent storage via localStorage.
